Relationships

From To Cardinality Key / Mechanism Notes
Cart Order 1 : Many Cart.sessionId → Order.session_id One shopping session can generate multiple orders (one per cart item)
Order Stock Many : 1 Order.pipe_type → Stock.pipe_type Stock decremented atomically on order creation (if qty ≤ 100)
Order ApprovedPrice Many : 1 (opt.) Order.pipe_type + region → ApprovedPrice Snapshot stored in Order; ApprovedPrice is admin override store
Node Backend ML Pricing Engine Many : 1 HTTP POST /calculate-price Called per /api/ai-price request; prev_month_sales aggregated from Order history
ML Pricing Engine RF Demand Model 1 : 1 (call) rf_model.predict(features) Random Forest trained on 9 features; output drives dynamic price factors
PK — Primary Key
FK — Foreign Key (logical ref in MongoDB)
UK — Unique Key / Index
Dashed border = external service (not in MongoDB)

Note: MongoDB uses embedded documents (CartItem inside Cart) instead of separate join tables. Relationships between collections are logical references by field value, not enforced foreign keys.
